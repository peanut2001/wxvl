#  思科防火墙零日漏洞已被恶意利用，用于部署 Interlock 勒索软件  
原创 网络安全9527
                    网络安全9527  安全圈的那点事儿   2026-03-19 05:47  
  
Interlock 勒索软件组织正在积极开展活动，利用 Cisco 安全防火墙管理中心 (FMC) 软件中的一个严重零日漏洞 (CVE-2026-20131)。  
  
该漏洞可能允许未经身份验证的远程攻击者在受影响的设备上以 root 权限执行任意 Java 代码。  
  
思科于2026年3月4日披露了该漏洞；该漏洞允许未经身份验证的远程攻击者以root权限执行任意Java代码。亚马逊威胁情报研究人员在漏洞公开披露前36天，即2026年1月26日，发现了Interlock利用该漏洞的攻击。  
  
这种先发优势使得勒索软件团伙能够趁防御者毫无察觉之际，积极攻击各类组织机构。亚马逊已将这些发现分享给思科，以支持其调查。AWS 基础设施和客户工作负载并未受到此次攻击的影响。  
  
调查取得进展，起因是配置错误的底层服务器暴露了Interlock公司的全部操作工具包。最初的威胁活动涉及向存在漏洞的软件路径发送HTTP请求，其中包含Java代码执行尝试和嵌入式URL。  
  
这些URL提供了配置数据，并通过触发HTTP PUT请求上传生成的文件来确认攻击成功。研究人员通过模拟被入侵的系统，诱使攻击者部署恶意Linux ELF二进制文件。  
  
暴露的测试服务器表明，该组织将工件整理成针对各个目标的专用路径，从而简化了工具的下载和被盗操作数据的上传。  
## 思科防火墙零日漏洞被利用  
  
技术指标可以肯定地将此次活动归因于 Interlock 勒索软件家族，这是一个以经济利益为目的的组织，于 2024 年 9 月首次出现。  
  
恢复的ELF二进制文件、嵌入的勒索信以及TOR谈判门户均与Interlock的既定品牌形象相符。他们的勒索信特意提及监管风险，以最大限度地向受害者施压，这符合他们已知的双重勒索模式。  
  
亚马逊威胁情报团队对时间戳的时间分析表明，攻击者活动区域位于UTC+3时区。Interlock历来针对那些运营中断会导致立即付款的行业，主要集中在教育、工程、建筑、制造、医疗保健和政府机构。  
  
Interlock 在获得访问权限后，会部署一套复杂的工具包来提升权限并保持持久性。恢复的 PowerShell 脚本会对 Windows 环境进行广泛的枚举，收集系统详细信息、浏览器痕迹和网络连接信息。  
  
该脚本将结果整理到每个主机的专用目录中，并将它们压缩成 ZIP 归档文件，从而为组织范围内的加密做好准备。  
  
该组织利用用 JavaScript 和 Java 编写的定制远程访问木马程序。其中，JavaScript 植入程序使用 Windows 管理规范 (WMI) 进行性能分析，并通过 RC4 加密消息建立持久的 WebSocket 连接。  
  
它提供交互式 shell 访问、文件传输和SOCKS5 代理功能。基于 GlassFish 库构建的功能相同的 Java 后门确保了冗余访问。  
  
为了掩盖踪迹，攻击者部署了一个 Bash 脚本，将 Linux 服务器配置为 HTTP 反向代理。该脚本安装 HAProxy 来转发流量，并每隔五分钟强制清除一次日志。  
  
此外，一个无文件、驻留在内存中的 Java webshell 会拦截包含 AES-128 加密命令的 HTTP 请求，该请求使用硬编码的种子。  
  
Interlock 还滥用合法工具，包括 ConnectWise ScreenConnect、用于内存取证的 Volatility 和用于 Active Directory 漏洞利用的 Certify，以及其自定义恶意软件。  
  
使用 Cisco 安全防火墙管理中心的组织必须立即应用最新的安全补丁。由于攻击者针对每个目标网络对下载的文件进行了高度定制，因此传统的基于特征码的检测方法已基本失效。  
  
相反，防御者应该专注于识别行为模式、内存驻留异常以及与 Interlock 多方面攻击链相关的特定网络侦察策略。
