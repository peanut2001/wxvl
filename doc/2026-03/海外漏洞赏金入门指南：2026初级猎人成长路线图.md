# BountyPath  
  
在 2026 年搜“漏洞赏金（Bug Bounty）”，你会看到两种极端观点：“网安已死/漏洞赏金到底谁在挖？真的难爆了！”或者“一个漏洞10万美元，比工资香多了”。  
  
这两种说法都具有误导性。虽然现在的漏洞赏金确实比之前更难，但它远未消亡。早期只需运行扫描器就能拿奖金，而今天，企业的防御更成熟，自动化扫描很多企业已经内部安全自检过了。  
  
**2026 年漏洞赏金的现状：**  
  
虽然企业的安全水平提高了，但攻击面却呈爆炸式增长。微服务、复杂 API、AI和多云架构的上线速度远快于其安全防护速度。 如果你愿意专注、持续学习、享受信息收集并保持耐心，漏洞赏金仍然值得投入。  
## 一、 前提条件  
  
在开始挖掘之前，你需要先打好基础。  
### 1. 基础网络知识  
  
你需要理解数据如何在互联网上流动：  
- DNS、IP 地址和端口。  
- 域名与子域名的工作原理。  
- 客户端与服务器端的区别。  
### 2. Web 应用基础  
- 熟练掌握 HTTP 请求与响应。  
- HTTP 方法（GET、POST、PUT、DELETE）。  
- Header、Cookie、Session 和 Token 的作用。  
- 状态码（200、403、404、500 等）。  
### 3. 基础编程意识  
  
不需要成为全栈开发，但必须能：  
- 阅读并理解基础 JavaScript。  
- 从高层次理解后端逻辑。  
- 修改参数并观察其影响。  
## 二、 选择你的第一个目标  
  
许多新手因为选择了错误的目标（如 Google、Meta 等热门项目）而受挫。  
  
**初学者的最佳策略：**  
1. **优先考虑 VDP（漏洞披露计划）：**  
  
1. 无奖金 → 竞争小。  
1. 更容易找到真实漏洞。  
1. 建立声望（进入名人堂/新人榜）对早期建立信心/正反馈很有帮助。  
1. **使用 Google Dorks 寻找人迹罕至的计划：**  
  
1. inurl:/security  
1. inurl:security.txt  
1. inurl:/responsible-disclosure  
1. **目标选择建议：**  
  
1. 选择更新或不太热门的src。  
1. 选择作为普通用户也能理解的应用逻辑。  
1. Web 应用和 API 而不是静态站点。  
## 三、 初学者侦察方法论  
  
盲目扫描只会导致重复报告。以下是一个高效的步骤：  
### 第一步：理解范围  
  
不要跳过阅读平台和项目方的公告！搞清楚哪些域名在范围内，哪些严禁触碰。  
### 第二步：子域名发现  
  
使用多源方法：  
- **被动收集：**  
 使用 Assetfinder  
、Subfinder  
 和 Findomain  
。  
- **数据去重：**  
 使用 anew  
 将结果合并到唯一列表中。  
### 第三步：存活探测与视觉检查  
- 使用 httprobe  
 过滤出真正运行的 Web 服务器。  
- 使用 Aquatone  
 对子域名进行截图，通过视觉快速筛选出有趣的登录页面、403 页面或过时的 UI。  
### 第四步：技术栈分析与自动化  
- **Wappalyzer：**  
 查看 CMS、语言和服务器版本。  
- **自动化三剑客：**Nuclei  
（扫描已知漏洞）、Nmap  
（开放端口检查）、Nikto  
（寻找敏感文件）。  
## 四、 适合新手的常见漏洞类型  
  
不要试图一次性掌握所有复杂的漏洞类型，建议从以下这些逻辑直观、易于验证的类型开始：  
1. **IDOR（越权）：**  
  
1. 场景：你登录后看到个人资料地址是 /api/v1/user/1234  
。  
1. 测试：将 ID 改为 1235  
。如果返回了他人数据，就是 IDOR。  
1. 技巧：创建两个账号（A 和 B）来验证越权漏洞。  
1. **XSS（跨站脚本攻击）：**  
  
1. 重点关注**反射型 XSS**  
。  
1. 在搜索框、“联系我们”表单或 URL 参数中尝试简单的 Payload，如 <u>Test</u>  
。  
1. **信息泄露与敏感文件：**  
  
1. 寻找 .env  
 文件（含 API 密钥）、.git  
 目录或 phpinfo()  
 页面。  
1. **子域名接管：**  
  
1. 检查指向已失效第三方服务（如 GitHub Pages、Heroku）的 DNS 记录。  
## 五、 实用的 60 天新手路线图  
### 阶段 1：打基础（第 1-2 周）  
- **目标：**  
 理解 Web 的真实运行机制。  
- **任务：**  
 学习 HTTP 协议、JavaScript 基础、网络知识。  
- **练习：**  
 完成 PortSwigger 的“Web Security Academy”免费实验。  
### 阶段 2：掌握工具（第 3-4 周）  
- **目标：**  
 熟练使用常用工具。  
- **任务：**  
- **Burp Suite：**  
 精通 Repeater、Intruder 和 Proxy。  
- **侦察工具：**  
 练习使用 subfinder  
 和 httpx  
。  
- **搜索引擎：**  
 学习 Google Dorking 和 Netlas.io。  
### 阶段 3：深入研究漏洞（第 5-6 周）  
- **目标：**  
 成为 3-4 种漏洞类型的专家。  
- **任务：**  
 在 HackerOne 上阅读至少 50 份公开报告；在 TryHackMe 或 HackTheBox 上练习。  
- **产出：**  
 为每种漏洞类型编写一份手动检查清单。  
### 阶段 4：实战狩猎（第 7-8 周）  
- **目标：**  
 发现第一个真实漏洞。  
- **任务：**  
 选择一个 VDP 程序，每天至少专注 2 小时在同一个目标上。  
- **报告：**  
 如果发现问题，按照高标准编写漏洞报告。  
## 六、 常见误区：避免浪费数月时间  
1. **不要盲目复制 Payload：**  
 如果你不理解其原理，就永远无法绕过简单的过滤器。  
1. **不要急于使用自动化工具：**  
 2026 年，简单的脚本漏洞早已被企业自己扫光了。  
1. **不要频繁更换目标：**  
 最成功的猎人通常会在一个目标上待数周，直到比开发人员还了解其代码。  
1. **不要忽视业务影响：**  
 报告“请求头缺失”没有意义。始终要深究：攻击者能用这个漏洞做什么？  
## 结论：希望自己能早点知道的建议  
  
漏洞赏金不是在座最聪明的人才能挖到，而在于你是是不是那个**最执着的人**  
。  
- **基础比什么都重要：**  
 你无法入侵你不理解的东西。  
- **名声比赏金更重要：**  
 早期专注于建立高质量的报告记录，过程认真，赏金会随之而来。  
- **多交流：**  
 加入 Discord 社区，在 X上关注安全研究员的活人分享。  
遵循这个路线图，保持专注，期待在排行榜上看到你的名字。
