#  微软确认 Windows 11 24H2/25H2 版本漏洞导致无法访问系统盘 C 盘  
原创 网络安全9527
                    网络安全9527  安全圈的那点事儿   2026-03-14 05:58  
  
微软已正式承认部分三星设备上的 Windows 11 用户存在一个严重漏洞，该漏洞会导致用户在安装2026 年 2 月安全更新后，系统盘 (C:) 完全无法访问。目前，微软正与三星积极合作，调查此问题。  
  
安装2026 年 2 月星期二补丁更新（KB5077181，操作系统版本 26100.7840）后，受影响的三星设备显示错误消息：“C:\ 无法访问 - 访问被拒绝。”  
  
该错误实际上会将用户锁定在自己的系统驱动器之外，阻止应用程序的启动，包括 Microsoft Outlook、Office 应用程序、Web 浏览器、系统实用程序和快速助手。  
  
该故障并非由任何特殊操作触发。诸如打开文件、启动应用程序或执行标准管理操作等常规任务就足以导致此错误。  
  
在严重的情况下，由于权限失败导致的连锁反应，用户甚至无法提升权限、卸载更新或收集诊断日志，这使得在没有外部支持的情况下几乎不可能进行自我修复。  
  
微软确认该问题主要出现在三星 Galaxy Book 4 和其他三星消费级设备上。包括巴西、葡萄牙、韩国和印度在内的多个地区都出现了相关报告。受影响的 Windows 版本为 Windows 11 版本 25H2 和 Windows 11 版本 24H2，目前服务器平台暂不受影响。  
  
企业管理员也在Reddit 的 r/sysadmin 社区中指出了这个问题，他们注意到，即使使用管理员凭据，Active Directory 域中受影响的 Galaxy Book 笔记本电脑也无法修改 NTFS 权限。  
  
现场报告的次要故障包括触控板驱动程序故障和无法打开 PowerShell，因为它的二进制文件位于现在无法访问的 C: 驱动器上。  
  
微软最新的调查指出，三星Share应用程序可能是造成此次事件的因素之一，但根本原因尚未得到完全证实。  
  
该公司表示，正在与三星合作，以确定故障是出在Windows更新本身，还是出在受影响设备上的三星软件生态系统。据信，该问题与系统驱动器根目录上的访问控制列表（ACL）损坏或应用错误有关。  
  
截至2026年3月13日，微软已将此问题列为“调查中”，尚未发布官方修复方案或临时解决方案。一位自称是巴西三星技术人员的Reddit用户发布了一个非官方的临时解决方案，即将C盘的所有权重新分配给“Everyone”组。但微软安全专家强烈警告不要采用这种方法，因为它会移除Windows内置的安全保护，使其无法保护关键系统目录。
