**PART****0****1**  
  
  
**漏洞情报**  
  
  
**1.Nginx UI信息泄露漏洞安全风险通告**  
  
  
3月9日，奇安信CERT监测到官方修复Nginx UI信息泄露漏洞(CVE-2026-27944)，该漏洞源于/api/backup端点无需身份验证即可访问，并在X-Backup-Security响应头中泄露了解密备份所需的加密密钥。攻击者能够下载服务器敏感数据（用户凭据、会话令牌、SSL 私钥、Nginx 配置）的完整系统备份并解密。奇安信鹰图资产测绘平台数据显示，该漏洞关联的国内风险资产总数为8938个，关联IP总数为1496个。目前该漏洞PoC和技术细节已公开。鉴于该漏洞影响范围较大，建议客户尽快做好自查及防护。  
  
  
**PART****0****2**  
  
  
**新增在野利用**  
  
  
**1.n8n 远程代码执行漏洞(CVE-2025-68613)******  
  
  
3月11日，CISA 将影响 n8n 的一个严重安全漏洞添加到其已知利用漏洞 ( KEV ) 目录中，原因是该漏洞已被积极利用。  
  
该漏洞编号为 CVE-2025-68613（CVSS 评分：9.9），涉及表达式注入，可导致远程代码执行。n8n 已于2025年12月在 1.120.4、1.121.1 和 1.122.0 版本中修复了此安全漏洞。CVE-2025-68613 是首个被列入 KEV 漏洞目录的 n8n 漏洞。  
  
美国网络安全和基础设施安全局 (CISA) 表示：N8n 的工作流表达式评估系统中存在动态管理代码资源控制不当的漏洞，该漏洞允许远程代码执行。据工作流自动化平台维护者称，经过身份验证的攻击者可以利用此漏洞，以 n8n 进程的权限执行任意代码。成功利用该漏洞可能导致实例完全被攻陷，使攻击者能够访问敏感数据、修改工作流程或执行系统级操作。  
  
目前尚无关于该漏洞在实际环境中如何被利用的详细信息。Shadowserver 威胁监控平台的数据显示，截至2026年2月初，有超过 24,700 个未修补的实例暴露在网络上，其中超过 12,300 个位于北美，7,800 个位于欧洲。  
  
该问题已在 n8n v1.122.0 版本中正式解决。强烈建议技术团队升级到 1.122.0 或更高版本，因为这些版本引入了必要的安全措施来限制表达式的求值方式。如果无法立即进行升级，管理员应部署以下短期临时解决方案：1.限制访问权限：严格限制工作流的创建和编辑权限，仅限完全信任的用户；2.强化环境：部署 n8n 时限制操作系统权限和网络访问权限，以控制潜在攻击的影响范围。

https://thehackernews.com/2026/03/cisa-flags-actively-exploited-n8n-rce.html  
  
  
**2.Ivanti Endpoint Manager 认证绕过漏洞(CVE-2026-1603)******  
  
  
3月9日，CISA 将 Ivanti Endpoint Manager (EPM) 的一个高危漏洞标记为正在遭受攻击。  
  
Ivanti 通过遍布全球的 7,000 多家合作伙伴网络，为 40,000 多家公司提供系统和 IT 资产管理产品。Ivanti 的 EPM 软件是一款一体化终端管理解决方案，可用于管理 Windows、macOS、Linux、Chrome OS 和 IoT 平台上的客户端设备。该安全漏洞被追踪为 CVE-2026-1603，远程威胁行为者无需任何权限即可利用此漏洞绕过身份验证，并在无需用户交互的低复杂度跨站脚本攻击中窃取凭据数据。Ivanti 一个月前发布了 Ivanti EPM 2024 SU5，修复了该漏洞，同时还修复了一个 SQL 注入漏洞，该漏洞允许远程经过身份验证的攻击者从数据库中读取任意数据。  
  
Ivanti 在最初的公告中表示：在公开披露之前，尚未发现任何客户受到这些漏洞的侵害。这些漏洞是通过披露计划披露的。目前，Shadowserver 威胁监控平台追踪着超过 700 个面向互联网的 Ivanti EPM 实例，其中大部分位于北美。然而，尚无信息表明其中有多少实例仍然容易受到 CVE-2026-1603 漏洞的攻击。  
  
虽然 Ivanti 尚未发现 CVE-2026-1603 漏洞被利用的证据，但威胁行为者经常在攻击中针对 Ivanti EPM 漏洞。建议受影响客户将产品升级到 Ivanti Endpoint Manager 2024 SU5 或更高版本。

**1.美国医疗设备巨头遭史诗级数据擦除：公司完全停摆、大量员工电脑手机被清空**  
  
  
3月12日The Record消息，美国医疗设备制造巨头史赛克（Stryker）透露，一次网络攻击导致公司微软环境“全球性网络中断”。据多名员工表示，许多服务器被清空，几乎所有工作应用无法使用，员工的工作电脑手机也被清空，公司业接近完全停摆。伊朗黑客组织Handala声称对此负责，称这是对美国导弹打击伊朗学校及美以对伊作战的报复。该组织表示已清除了超20万个服务器、系统和移动设备，窃取了超50TB的内部数据。史赛克声明称，已控制事件态势，并部署了业务连续性措施。

https://therecord.media/stryker-cyberattack-iran-hackers  
  
  
**2.麦肯锡AI助手被红队AI攻陷：近5000万条聊天记录可任意访问 涉海量客户机密数据**  
  
  
3月9日The Register消息，美国安全厂商CodeWall的研究人员表示，他们的AI智能体成功入侵了麦肯锡的内部AI平台Lilli，并在仅仅2小时内获得了该聊天机器人的完整读写访问权限，能够访问4650万条明文存储的聊天消息，其中涉及海量客户机密信息。CodeWall公司表示，整个过程“从目标研究、分析、攻击到报告，完全由智能体自主完成”。智能体发现Lilli有22个API未设身份验证，可通过某个API利用SQL注入漏洞访问数据库，进而获取包含系统提示的敏感信息。研究人员指出，该漏洞支持读写操作，攻击者可篡改Lilli的系统提示，污染聊天机器人的回答内容、安全护栏及引用来源，且攻击过程仅需一次HTTP调用，无需代码修改或部署。  
  
  
原文链接：  
  
https://www.theregister.com/2026/03/09/mckinsey_ai_chatbot_hacked/  
  
  
**3.爆红AI工具OpenClaw暗藏高危风险，国内多个部门发布风险提示**  
  
  
3月上旬安全内参综合消息，近期，OpenClaw（也称“龙虾”） 智能体软件在全球迅速走红，国内更是掀起“养龙虾”热潮。国家互联网应急中心、中央网信办数据与技术保障中心、国家信息安全漏洞库、工信部网络安全威胁和漏洞信息共享平台、国家工业信息安全发展研究中心等连续发布关于OpenClaw智能体的安全风险提示。OpenClaw依据自然语言指令直接操控计算机完成相关操作，由于不当安装和使用，当前已经出现了一些严重的安全风险，包括提示词注入、误操作、Skills投毒、安全漏洞等，相关单位和个人用户要注意防范潜在的各类安全风险。据悉，目前已有多家金融机构、高校发文，要求内部禁止使用OpenClaw。

**1.美国白宫发布2026版《美国网络战略》**  
  
  
3月6日，美国白宫发布《特朗普总统的美国网络战略》（President Trump’s Cyber Strategy for America）。该文件以“维持美国在网络空间的绝对领先地位”为目标，通过六大行动支柱，构建起攻防一体、政企协同、技术封锁的网络战略体系。六大行动支柱包括：塑造对手行为，强调部署进攻性网络能力，并将私营部门纳入进攻体系；倡导常识性监管，要求精简合规要求，增强敏捷性、激发创新力；现代化和保护联邦政府网络，提出同步推进抗量子密码、零信任、云迁移与人工智能四类技术，赋能联邦网络防护；保护基础设施安全，要求确保信息与运维技术供应链安全，剔除对手国家的供应商和产品；维持关键和新兴技术领域优势，重点推动人工智能在网络安全领域的创新，快速部署代理式人工智能等赋能工具；培育人才与能力，建立人才培养与共享渠道。
