## 0x00项目整体架构与意义  
  
xingrin致力于解决网络安全中资产发现与漏洞扫描的关键问题。随着企业网络规模的不断扩大，资产数量日益增多且分布复杂，传统的手动资产梳理和漏洞检测方式效率低下且容易遗漏。xingrin通过自动化的手段，整合多种功能，旨在帮助用户快速、准确地发现网络资产，并及时检测出其中存在的安全漏洞，为网络安全防护提供有力支持。  
## 0x01功能特性详解  
### 0x0101目标与资产管理  
1. **组织管理**  
：多层级目标组织架构为企业提供了高度灵活的资产管理方式。不同规模和业务结构的企业，其网络资产的组织方式各不相同。xingrin的多层级组织管理允许企业根据自身需求，将资产进行细致的分组。例如，大型企业可能按照部门、地域等维度进行分层管理，小型企业则可以根据业务类型进行简单分组。这种灵活性使得企业能够更好地梳理和掌控自身的网络资产，提高管理效率。  
  
1. **目标管理**  
：支持域名和IP目标类型，涵盖了网络资产的两种主要标识方式。无论是基于域名访问的网站、应用程序，还是通过IP地址直接连接的服务器、网络设备等，xingrin都能将其纳入管理范围。这确保了企业的各类网络资产都能被有效识别和监控，为后续的资产发现和漏洞扫描奠定基础。  
  
1. **资产发现**  
：具备子域名、网站、端点、目录自动发现功能。在网络环境中，许多潜在的安全风险隐藏在这些不易察觉的资产中。通过自动化的发现机制，xingrin能够快速挖掘出这些隐藏资产。例如，子域名可能由于管理疏忽而存在安全漏洞，自动发现功能可以及时找到这些子域名，并纳入扫描范围。对于网站、端点和目录的发现，有助于全面了解网络应用的架构和资源分布，为漏洞扫描提供更全面的信息。  
  
1. **资产快照**  
：扫描结果快照对比功能是xingrin的一大特色。随着时间推移，网络资产会不断发生变化，新的功能上线、旧的服务更新等都可能带来新的安全风险。通过对不同时间点的扫描结果进行快照对比，用户可以清晰地追踪资产的变化情况。例如，发现某个目录下新增了文件，或者某个端点的访问权限发生了改变，及时分析这些变化对网络安全的影响，提前预防潜在的安全威胁。  
  
### 0x0102漏洞扫描  
1. **多引擎支持**  
：集成Nuclei等主流扫描引擎，充分利用了不同扫描引擎的优势。Nuclei以其高效、灵活的扫描能力而闻名，能够检测多种类型的漏洞。通过集成多个主流扫描引擎，xingrin可以更全面、准确地检测网络资产中的漏洞。不同的扫描引擎在检测不同类型漏洞时可能具有不同的优势，例如某些引擎对Web应用漏洞检测更擅长，而另一些则在系统漏洞检测方面表现出色，多引擎的集成使得漏洞检测更加全面。  
  
1. **自定义流程**  
：采用YAML配置扫描流程，为用户提供了极大的灵活性。YAML语言简洁易读，用户可以根据自身网络资产的特点和安全需求，灵活编排扫描流程。例如，对于特定的业务系统，用户可以定制先检测哪些类型的漏洞，以及按照何种顺序进行扫描。这种自定义能力使得xingrin能够适应不同企业的个性化安全需求，提高漏洞扫描的针对性和效率。  
  
1. **定时扫描**  
：利用Cron表达式配置自动化周期扫描，帮助企业实现常态化的安全检测。网络安全威胁随时可能出现，定期进行漏洞扫描是保障网络安全的重要手段。通过Cron表达式，用户可以精确设置扫描的时间间隔，如每天、每周或每月的特定时间进行扫描。这样，企业无需手动干预，就能定期获取网络资产的安全状况报告，及时发现并处理新出现的漏洞。  
  
### 0x0103分布式架构  
1. **多节点扫描**  
：支持部署多个Worker节点，实现了扫描能力的横向扩展。随着企业网络资产规模的增长，单个扫描节点可能无法满足快速、全面扫描的需求。通过增加Worker节点，xingrin可以同时对多个资产进行扫描，大大提高扫描效率。例如，在大型企业网络中，可能有成千上万个网络资产需要扫描，多节点扫描能够显著缩短扫描时间，及时发现潜在的安全风险。  
  
1. **本地节点**  
：零配置，安装即自动注册本地Docker Worker，降低了使用门槛。对于一些技术能力有限或希望快速部署的用户来说，无需复杂的配置步骤，只需安装xingrin，本地的Docker Worker就能自动注册并投入使用。这使得企业能够快速搭建起扫描环境，开始进行资产发现和漏洞扫描工作。  
  
1. **远程节点**  
：通过SSH一键部署远程VPS作为扫描节点，进一步拓展了扫描的范围和能力。对于一些需要对远程网络资产进行扫描的场景，或者希望利用云端资源提高扫描效率的用户，远程节点的部署提供了便捷的解决方案。用户只需通过简单的SSH命令，就能将远程VPS配置为扫描节点，纳入到xingrin的扫描体系中。  
  
1. **负载感知调度**  
：实时感知节点负载，自动分发任务到最优节点，确保扫描任务的高效执行。在多节点扫描环境中，不同节点的负载情况可能会随时变化。如果任务分配不合理，可能导致某些节点负载过高，而其他节点闲置。xingrin的负载感知调度功能能够实时监测各个节点的CPU、内存等资源使用情况，将扫描任务自动分配到负载较低的最优节点，提高整个系统的扫描效率。  
  
1. **节点监控**  
：实时心跳检测，以及对CPU、内存、磁盘状态的监控，让用户随时了解节点的运行状况。通过实时心跳检测，系统能够及时发现节点是否离线，确保扫描任务的连续性。而对CPU、内存、磁盘状态的监控，则有助于用户及时发现节点资源使用异常情况，提前进行调整和优化，保障扫描工作的稳定进行。  
  
1. **断线重连**  
：节点离线自动检测，恢复后自动重新接入，进一步增强了系统的稳定性。在网络环境中，节点可能由于网络故障、硬件问题等原因暂时离线。xingrin的断线重连功能能够自动检测到节点离线情况，并在节点恢复正常后，自动重新接入系统，继续执行扫描任务，减少因节点离线导致的扫描中断时间。  
  
## 0x02一键安装与常用命令  
### 0x0201一键安装  
1. **克隆项目**  
：  
  
```
git clone https://github.com/yyhuni/xingrin.git
cd xingrin
```  
  
通过这两条命令，用户可以从GitHub上克隆xingrin项目的代码到本地，并进入项目目录。这是安装的第一步，获取项目的源代码是后续安装和运行的基础。 2. **安装并启动（生产模式）**  
：  
```
sudo./install.sh
```  
  
执行该命令后，系统将按照预设的安装脚本进行安装，并以生产模式启动xingrin。生产模式通常针对正式的生产环境进行了优化，确保系统的稳定性和性能。在安装过程中，脚本可能会自动安装所需的依赖项，配置相关参数，为xingrin的正常运行做好准备。  
### 0x0202访问服务  
  
安装完成后，用户可以通过Web界面访问xingrin，地址为https://ip:8083  
。这里的“ip”是指安装xingrin的服务器IP地址。通过Web界面，用户可以方便地进行目标与资产管理、漏洞扫描配置、查看扫描结果等操作，提供了一个直观、便捷的管理平台。  
### 0x0203常用命令  
1. **启动服务**  
：  
  
```
sudo./start.sh
```  
  
当需要启动xingrin服务时，执行该命令。这对于在系统重启后或者服务意外停止的情况下重新启动服务非常有用。 2. **停止服务**  
：  
```
sudo./stop.sh
```  
  
如果需要停止xingrin服务，例如进行系统维护、升级等操作，可以执行此命令。它会按照一定的流程关闭相关的进程和服务，确保系统资源的正确释放。 3. **重启服务**  
：  
```
sudo./restart.sh
```  
  
在对xingrin进行一些配置更改后，可能需要重启服务使更改生效。执行该命令可以先停止服务，然后再重新启动，确保新的配置能够正常应用。 4. **卸载**  
：  
```
sudo./uninstall.sh
```  
  
当不再需要使用xingrin或者需要重新安装时，可以执行卸载命令。该命令会删除相关的文件、配置和进程，将系统恢复到安装前的状态。 5. **更新**  
：  
```
sudo./update.sh
```  
  
为了保持xingrin的功能和安全性，及时更新是很重要的。执行更新命令后，系统会从项目的官方源获取最新的代码和更新内容，并进行相应的安装和配置，确保用户能够使用到最新版本的xingrin及其新功能。
