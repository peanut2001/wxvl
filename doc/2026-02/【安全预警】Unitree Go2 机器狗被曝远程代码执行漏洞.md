**一、基本信息**  
  
  
**漏洞编号**  
：CVE-2026-27509 & CVE-2026-27510  
  
**漏洞类型**  
：远程代码执行（Remote Code Execution）  
  
**影响产品**  
：Unitree Go2系列四足机器人  
  
**危害等级**  
：严重（Critical）  
  
**披露状态**  
：已公开  
  
**厂商响应**  
：已确认  
  
  
**二、漏洞概述**  
  
  
**CVE-2026-27509**  
：该漏洞源于 Unitree Go2 在特定固件版本中对 DDS（Data Distribution Service）通信缺乏有效的身份验证与消息完整性校验。攻击者只要处于相邻网络环境，即可构造恶意 DDS 消息向机器人下发并注册自定义 Python 代码，并可将其绑定至物理控制器按键触发执行。代码以 root 权限运行，且绑定关系可跨重启保留，构成高危远程代码执行（RCE）风险。  
  
**DDS（Data Distribution Service）**  
 是一种面向实时系统的 **发布-订阅（Publish/Subscribe）通信中间件标准**  
，广泛用于机器人、自动驾驶、工业控制、无人机和军工系统等对低延迟、高可靠性要求极高的场景。DDS本质上是一个智能消息总线，与MQTT类似，组件之间不再直接通过API调用，而是将消息发布到 Topic，感兴趣的组件可以订阅这些 Topic。  
  
**CVE-2026-27510**  
：该漏洞核心问题在于 Go2 Android应用将用户自定义程序存储在本地SQLite数据库中，但在传输到机器人执行前未进行充分的完整性与安全校验。攻击者可通过本地访问 Android 设备并修改数据库，或者通过社区下载的恶意程序导入机器人，在机器人端触发执行并获得 root 权限。由于恶意程序可被持久保存或绑定控制逻辑，该漏洞同样具备持久化控制风险。  
  
Unitree 的智能机器人平台在12月13日开放了机器人应用市场UniStore，允许用户上传、下载和分享机器人程序。在安全事件曝光后，再次打开应用市场UniStore，发现所有的动作库都已经被隐藏。原本热门“扭扭舞”、“李小龙”等动作模板都无法找到。  
  
  
**三、安全趋势警示：为什么智能机器人总是不安全**  
  
  
从既往安全事件披露情况来看，这并不是第一次出现严重漏洞问题。去年，海外安全研究者Bin4ry（Andreas Makris）与KF（Kevin Finisterre）在GitHub发布了名为UniPwn的项目，披露了宇树机器人/机器狗蓝牙服务中存在的一个可导致设备被攻破的命令注入漏洞。  
  
**一个残酷的事实是：在具身智能领域，安全往往是最后一个被考虑的问题。**  
  
产品导向对安全导向的挤压，是智能机器人行业频繁出现漏洞的重要因素。在市场竞争激烈的环境中，企业通常更关注产品功能迭代速度和性能创新。研发周期被压缩，安全团队规模相对有限，安全测试时间不足。在这种情况下，发布前的安全审查往往难以覆盖所有风险，“先上线、再修补”的策略也容易成为常态。  
  
行业标准的缺失进一步加剧了安全隐患。与汽车行业的碰撞测试或一般电子产品的3C认证相比，智能机器人尚未形成统一的安全评估和认证体系。企业缺乏强制的安全更新义务，也没有统一的漏洞披露规范或事故责任认定机制。在这种监管真空下，安全更多依赖企业自律，风险因而容易被放大。  
  
  
**四、当未来 AI 机器人走进生活：安全漏洞如何影响长期信任**  
  
  
随着具身智能的发展，未来的 AI 机器人将不再只是执行简单动作的设备，而是具备持续学习能力、环境理解能力与自主决策能力的智能体。它们可能承担家庭陪护、儿童陪伴、安防巡检、医疗辅助甚至情绪交互等角色。在这种场景下，机器人不只是工具，而是高度融入私人空间的数字存在。  
  
在这种趋势下，安全漏洞的影响将呈现新的特征。远程代码执行不再只是系统被入侵，而是意味着：机器人可以被外部操控、行为逻辑被篡改、环境感知被窃取。摄像头画面、语音交互记录、空间结构数据，都可能成为攻击者获取的目标。更重要的是，机器人具备主动移动能力，一旦失控，其风险不再局限于数据层面，而是可能扩展至现实空间。  
  
传统电脑或手机被攻击，用户可以关机、断网、重装系统；而未来的 AI 机器人是会在客厅中移动、在卧室中充电、在身边互动的设备。一旦用户意识到它可能被远程操控，即便漏洞已经修复，心理上的不确定感仍然可能持续存在。设备是否完全安全？是否仍有未知风险？这种持续的不确定性，本身就可能影响使用体验与信任基础。尤其是在家庭场景中，一旦用户对其安全性产生怀疑，原本象征科技便利的产品，可能转化为潜在风险源。这种认知转变，会影响用户的安全感与使用体验，甚至影响对整个智能设备生态的信任。  
  
从产业角度看，安全漏洞不仅是技术问题，更是信任管理问题。一次高危漏洞披露，可能影响的不只是设备销量，还包括用户对品牌、对行业乃至对智能化生活方式的整体信心。  
  
因此，在具身智能时代，安全不仅关乎代码质量，也关乎用户心理预期的稳定性。只有当安全治理机制成熟、漏洞响应透明、补丁发布及时，用户的信任才可能长期维持。  
  
  
**五、防护措施**  
  
  
**固件升级**  
：更新到最新官方补丁版本（如 v1.1.13 或更高）。通告显示 Unitree 已修复部分漏洞并发布补丁。  
  
**移动应用升级**  
：确保 Android 应用为最新版本，并从官方渠道安装。  
  
**避免使用不可信程序**  
：不要从社区导入未经验证的程序，这会是漏洞触发的主要途径之一。如果已经从非官网来源导入过程序，应当恢复出厂设置避免设备被持久化利用。
