1. IBM Concert 信任管理问题漏洞（CNNVD-202602-2640）****  
  
IBM Concert是美国国际商业机器（IBM）公司的一种新工具，使用生成式AI来帮助管理复杂的云原生应用程序。  
  
IBM Concert 1.0.0版本至2.1.0版本存在信任管理问题漏洞，该漏洞源于使用硬编码用户凭据，攻击者利用该漏洞可以获取敏感信息。  
  
目前厂商已发布升级补丁以修复漏洞，参考链接：  
  
https://www.ibm.com/support/pages/node/7260162  
  
2. Microsoft Windows Admin Center 授权问题漏洞（CNNVD-202602-2604）****  
  
Microsoft Windows Admin Center是美国微软（Microsoft）公司的一款本地部署的基于浏览器的应用程序，具有管理服务器、群集等功能。  
  
Microsoft Windows Admin Center存在授权问题漏洞，该漏洞源于身份验证不当，攻击者利用该漏洞可以通过网络提升权限。  
  
目前厂商已发布升级补丁以修复漏洞，参考链接：  
  
https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-26119  
  
3. WordPress plugin IDonate 授权问题漏洞（CNNVD-202602-3273）****  
  
WordPress和WordPress plugin都是WordPress基金会的产品。WordPress是一套使用PHP语言开发的博客平台。该平台具有在基于PHP和MySQL的服务器上架设个人博客网站的功能。WordPress plugin IDonate是一个应用插件。  
  
WordPress plugin IDonate 2.1.5版本至2.1.9版本存在授权问题漏洞，该漏洞源于缺少能力检查，攻击者利用该漏洞可以提升权限。  
  
目前厂商已发布升级补丁以修复漏洞，参考链接：  
  
https://wordpress.org/plugins/idonate/

1. MLflow 安全漏洞（CNNVD-202602-3329）****  
  
MLflow是一个开源的简化机器学习开发的平台，具有跟踪实验、将代码打包成可重复运行以及共享和部署模型的功能。  
  
MLflow存在安全漏洞，该漏洞源于使用硬编码默认凭据，攻击者利用该漏洞可以任意执行代码。  
  
目前厂商已发布升级补丁以修复漏洞，参考链接：  
  
https://mlflow.org/releases  
  
2. OpenClaw 安全漏洞（CNNVD-202602-3715）****  
  
OpenClaw是一个开源的智能人工助理。  
  
OpenClaw 2026.2.15之前版本存在安全漏洞，该漏洞源于Docker工具沙箱配置注入问题，攻击者利用该漏洞可以导致容器逃逸或访问主机数据。  
  
目前厂商已发布升级补丁以修复漏洞，参考链接：  
  
https://github.com/openclaw/openclaw/releases  
  
3. OpenSift 安全漏洞（CNNVD-202602-3800）****  
  
OpenSift是一款开源的人工智能学习助手。  
  
OpenSift 1.1.2-alpha版本及之前版本存在安全漏洞，该漏洞源于在UI界面中使用不安全的HTML模式渲染不受信任的用户或模型内容，攻击者利用该漏洞可以执行JavaScript代码。  
  
目前厂商已发布升级补丁以修复漏洞，参考链接：  
  
https://github.com/OpenSift/OpenSift/releases/
